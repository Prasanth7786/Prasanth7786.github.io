Elastic SIEM Setup and Threat Detection using Honeypots

Project Overview

This project demonstrates a complete SIEM deployment using the Elastic Stack (Elasticsearch, Logstash, Kibana) to monitor and analyze real-world cyber attacks. A combination of honeypots, including Cowrie (SSH honeypot) and a custom fake login page, were deployed to simulate attack surfaces. Filebeat was used to collect the logs from these honeypots and forward them to Elasticsearch. Kibana dashboards were created to visualize the data and SIEM detection rules were implemented to identify brute-force attacks.

Project Objectives

  • Deploy Elastic SIEM using the ELK stack to enable log analysis and threat detection
  • Simulate attack surfaces using Cowrie SSH honeypot and a custom web login form
  • Collect and forward logs with Filebeat for real-time analysis
  • Visualize attacker activity using Kibana dashboards
  • Set up detection rules and alerts to identify brute-force attempts and suspicious IP addresses

Tools and Technologies Used

Elastic Stack (ELK):

Elasticsearch Logstash Kibana

Security Tools:

Elastic SIEM Cowrie Honeypot Custom Fake Login Page Filebeat Kali Linux

System Setup

1. ELK Stack Deployment

  • Elasticsearch, Logstash, and Kibana were installed and configured on an Ubuntu virtual machine
  • Logstash pipelines were set up to parse incoming logs from Filebeat
  • Kibana was configured to visualize data and manage SIEM detection rules

ELK Stack deployment on Ubuntu VM

2. Honeypot Setup

  • Cowrie was configured to emulate an SSH server and capture login attempts
  • A Flask-based fake login page was created to log web-based brute-force attempts
  • Both honeypots were hosted on the same Ubuntu VM

Nmap scan results showing honeypot's open ports (SSH, HTTP)

3. Filebeat Configuration

  • Filebeat collected logs from Cowrie and the fake login page
  • Log files were parsed using custom modules and forwarded to Logstash
  • Logstash enriched and transformed the logs before storing them in Elasticsearch

Filebeat configuration showing log collection from Cowrie honeypot and fake login page

Threat Simulation and Detection

1. Brute-Force Attack Simulation

  • Hydra on Kali Linux was used to simulate brute-force attacks on:
    • Cowrie SSH honeypot
    • Flask-based fake login form
  • Logs generated by these attacks were captured and indexed by Elasticsearch

Hydra tool performing brute-force attack against the honeypot services

2. Alert Setup in Elastic SIEM

  • Custom detection rules were created to identify:
    • Multiple failed SSH login attempts
    • Multiple failed web login attempts
    • Unusual IP addresses accessing the system
  • Threshold-based alerts triggered when the count of failed attempts from a single IP crossed a limit

SIEM alerts showing SSH brute-force attempts detected from multiple source IPs

SIEM alerts showing brute-force attempts detected on the web login page

Visualization and Dashboards

Dashboards Included:

  • Top Attacking IP Addresses: Identified the most frequent source IPs of login attempts
  • Most Used Usernames: Showed commonly guessed usernames by attackers
  • Most Used Passwords: Displayed commonly attempted passwords
  • Login Attempts Over Time: Visualized brute-force activity over time

Kibana dashboard displaying most commonly used usernames and passwords in brute-force attempts

Visualization of top attacking IP addresses

Results

  • The system successfully detected and visualized brute-force attacks
  • All login attempts were recorded with metadata (username, password, source IP, timestamp)
  • Kibana dashboards provided insights into attacker behavior and trends
  • Elastic SIEM generated alerts in near real-time based on the detection rules

Conclusion

The project demonstrates the effective use of Elastic SIEM for monitoring, detecting, and visualizing cyber attacks on honeypots. With real-time log ingestion, enrichment, and analysis, the system provided actionable insights through alerts and dashboards. This hands-on experience solidified an understanding of deploying SIEM solutions and interpreting attacker behavior.
